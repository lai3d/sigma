Why Cilium for Sigma?

Advantages

  • eBPF synergy — Sigma agent already uses eBPF for traffic monitoring; Cilium uses eBPF for CNI/gateway/policy
  • Replaces kube-proxy entirely — pure eBPF networking, lower latency
  • Gateway API native support — no extra controller needed
  • CiliumNetworkPolicy — L3/L4/L7 identity-based, DNS-aware
  • Hubble — flow-level observability, service map, Grafana integration
  • WireGuard transparent encryption (optional, replaces mTLS)
  • XDP fast-path — kernel-bypass DDoS mitigation
  • No sidecar overhead (unlike Istio)
  • Bandwidth Manager (EDT) for traffic shaping

Considerations

  • Requires Linux 5.10+ kernel with eBPF support
  • CNI replacement — must be installed at cluster bootstrap (not retrofittable easily)
  • Heavier learning curve than simple ingress controllers
  • L7 policy uses embedded Envoy (adds some overhead)
  • Less mature Gateway API implementation vs Envoy Gateway
  • Managed K8s compatibility varies (EKS/GKE: good, some providers: limited)
  • Debugging eBPF programs is harder than iptables rules
  • Hubble relay adds resource overhead on each node
